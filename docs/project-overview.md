# Containerized Multi-Tier AWS Monitoring Project

## Project Overview

This project demonstrates deployment, monitoring, and alerting for a containerized application running on AWS EC2.

The primary objective is to simulate real-world Cloud Support, NOC, Infrastructure, and Junior Cloud Engineer responsibilities by implementing monitoring, alerting, troubleshooting, and operational visibility.

## Key Features

* EC2 Instance Deployment
* Docker Container Management
* CloudWatch Monitoring
* CloudWatch Agent Installation
* CPU Monitoring
* Memory Monitoring
* Disk Monitoring
* SNS Email Notifications
* CloudWatch Dashboards
* Incident Detection and Alerting

## Technologies Used

### Cloud Services

* AWS EC2
* Amazon CloudWatch
* Amazon SNS
* IAM

### Containerization

* Docker

### Operating System

* Ubuntu Server 22.04

## Project Goals

* Monitor infrastructure health
* Detect resource utilization issues
* Trigger alerts automatically
* Provide operational visibility
* Simulate production support workflows

## Outcomes

After completing this project, users can:

* Deploy workloads on AWS EC2
* Configure CloudWatch Agent
* Create CloudWatch Dashboards
* Configure CloudWatch Alarms
* Implement Email Notifications
* Troubleshoot Monitoring Issues
* Perform Basic Incident Response
